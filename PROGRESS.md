# Пати-игры — прогресс

Хаб компанейских мини-игр (заходят с телефонов в одну комнату по коду).
Стек как у «Шпиона»: чистая статика (HTML/CSS/vanilla JS) + Firebase RTDB, GitHub Pages, пуш через прокси 7897.

## Ростер (решено)
1. **Словесный движок** (тумблер: Совпади / Не совпади / На одной волне) — later.
2. **100 к 1** — ✅ MVP готов (эта итерация).
3. Одна лёгкая (Кто из нас / реакция) — later.

## Статус: MVP «Каркас + 100 к 1» собран и протестирован (кроме кликов в браузере)

### Тесты пройдены
- **Правила Firebase:** проверено REST-пробой через прокси — узел `/party` уже пишется (правила проекта открыты глобально, как у «Шпиона»). Публиковать ничего не нужно.
- **Матчинг:** 136/136 кейсов (перефраз/склонения/синонимы/опечатки ловятся, «вообще другое» — мимо).
- **Интеграция с живым Firebase** (`scratchpad/sim.js`): полный сценарий лобби+раунд против реальной БД —
  входы, «серый» presence, кик, ход по кругу, скип офлайн-хостом, матчинг, счёт (Аня 58/Боря 42/Вера 0),
  конец раунда, anti-peek, чтение из БД обратно, очистка. Всё зелёное.
- **UI-обвязка:** статическая сверка `app.js` ↔ `index.html` — все id/классы на месте.
- **НЕ покрыто:** клики в реальном браузере (нет инструмента автоматизации UI) — финальное звено
  «игрок пишет pending → хост разбирает» подтвердится при живом заходе с двух устройств.

### Что сделано
- **Каркас комнат** (`app.js`): создание/вход по 4-символьному коду, автоочистка старых комнат, автовосстановление сессии по localStorage-id.
- **Presence «серый»**: при обрыве игрок не дропается, гаснет серым (`online:false` через `onDisconnect`), при возврате — снова онлайн, слот/очки на месте. Heartbeat `lastSeen` раз в 25 c.
- **Хост**: кикает игроков; при обрыве ведущего на **15 мин** права уходят самому «старому» онлайн-игроку (транзакция).
- **100 к 1** (пошаговый вскрыв):
  - Хост стартует раунд → вопрос из базы (фильтр `soft`/`hot` тумблером «Пак 18+»), случайный без повтора прошлого.
  - Игроки по кругу пишут догадку; ход уходит в `round/pending`, **хост разбирает** (матчинг) и вскрывает ячейку, начисляет очки, передаёт ход. Пас — тоже через pending.
  - **Anti-peek**: полное табло ответов держит только хост (память + localStorage); в БД до вскрытия ячейки её текста/очков нет.
  - Две подсказки: 💡 сразу, 🔍 по кнопке хоста.
  - Ход офлайн-игрока ждём; у хоста кнопка «Пропустить ход».
  - Конец раунда: всё табло вскрыто / все спасовали / хост завершил → полное табло + счёт. Дальше «Следующий раунд» или «В лобби».
  - **Азарт v1** = таблица лидеров по очкам (без ставок — по решению).

### Матчинг «по смыслу» (проверен: 136/136 кейсов)
Пайплайн: нормализация (регистр, ё→е) → токены (слова 3+ без стоп-слов) → сравнение слов (точное / по основе / Левенштейн ≤1) → пересечение токенов догадки и ответа (текст + ключи-синонимы), порог 3.
- Ловит: перефраз («вечно про еду спрашивает»→«Ты поел?»), склонения (еду↔еда), синонимы (шаверма→шаурма), опечатки, односложный ввод («деньги»).
- Отсекает «вообще другое» (нет общих токенов → мимо).
- Известный предел: **отрицание** («никогда не начну» матчинг не видит «не»).
- Правило ключей: не класть в ключи слово из самого вопроса (иначе эхо-ложняки — вычищено).

## ТРЕБУЕТСЯ ОТ ВЛАДЕЛЬЦА (иначе запись в БД не пройдёт)
Опубликовать правила Realtime Database, добавив узел `party` (к существующим `rooms`/`pool`):
```json
{
  "rules": {
    "rooms": { ".read": true, ".write": true },
    "pool":  { ".read": true, ".write": true },
    "party": { ".read": true, ".write": true }
  }
}
```
(Правила открытые — как у «Шпиона»; для компанейской игры ок.)

## Как протестировать локально
1. `python serve.py` в папке проекта → открыть `http://localhost:8080`.
2. Вторую-третью «руку» — в других вкладках/инкогнито/с телефона в той же сети.
3. Создать комнату, зайти по коду, начать 100к1, проверить: ход по кругу, вскрытие, счёт, «серый» при закрытии вкладки, кик, подсказка.
