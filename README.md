# Пати-игры

Хаб компанейских мини-игр: заходишь с телефона в общую комнату по 4-символьному коду.
Чистая статика (HTML/CSS/vanilla JS) + Firebase Realtime Database, хостинг — GitHub Pages.

**Игры:** «100 к 1» (готово) · словесные (Совпади / Не совпади / На одной волне) и лёгкая — в планах.

## Запуск локально
```bash
python serve.py      # → http://localhost:8080
```
Вторую-третью «руку» открывай в инкогнито или с телефона в той же сети.

## Как играть в «100 к 1»
Ведущий создаёт комнату, остальные заходят по коду. Ведущий жмёт «Начать игру» —
на экране вопрос, игроки **по очереди** пишут догадку. Ответ засчитывается «по смыслу»
(не обязательно слово-в-слово). За вскрытый ответ — его очки. Побеждает набравший больше.

## Устройство
- `index.html`, `css/style.css` — интерфейс.
- `js/firebase-config.js` — конфиг Firebase (узел `/party`).
- `js/questions.js` — база вопросов (**автогенерация** из `БАЗА_вопросов.md` через `tools/parse_base.py`).
- `js/app.js` — движок: комнаты, присутствие, «100 к 1».
- `ПОРТРЕТ_ЦА.md`, `БАЗА_вопросов.md` — исследование ЦА и контент.
- `PROGRESS.md` — статус и решения.

## Пересобрать базу вопросов
После правок `БАЗА_вопросов.md`:
```bash
python tools/parse_base.py
```

## Firebase
Нужны открытые правила на узел `party` (`.read`/`.write: true`). Ключи в `firebase-config.js`
не секретны — доступ ограничивается правилами БД.
